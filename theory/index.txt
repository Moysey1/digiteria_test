1. 
Каким будет результат следующей операции на языке JavaScript: [1, 2, 3] + [4, 5, 6]. Почему был получен такой результат?

Ответ: 

Результатом будет строка "1,2,34,5,6", так как массивы будут преобразованы в строки, 
затем эти строки будут обьеденены в результате операции конкатенации строк, 
поэтому из преобразованных массивов "1,2,3" + "4,5,6", получиться строка "1,2,34,5,6". 

2. 
Дано выражение:
var a = {b: 1};
var b = a;
b.b = 2;
console.log(a);
Что будет выведено в консоли? Почему был получен такой результат?

Ответ:

В консоль будет выведено === { b: 2 }.

Это происходит из-за особенностей работы с объектами в JavaScript. 
Переменная (a) ссылается на объект { b: 1 }. Затем переменная (b) присваивается значению переменной (a), 
поэтому (b) также ссылается на этот объект. Когда мы изменяем значение свойства (b) через переменную (b), 
это изменение также отражается в объекте, на который ссылается переменная (a). 

3. 
Написать регулярное выражение, совпадающее с числом с плавающей точкой с точностью до 3 знака после запятой.

Ответ: 

Шаблон Вырвжения === (/^\d+\.\d{1,3}$/)

Расшифровка:

- `^` - начало строки
- `\d+` - одна или более цифр 
- `\.` - точка (для разделения целой и десятичной частей)
- `\d{1,3}` - от одной до трех цифр
- `$` - конец строки

Пример:
const isURL = str => true
isURL(‘https://example.com’)
isURL(‘http://www.example.com’)

Ответ: 

const isURL = str => {
  const regex = /^(?:https?:\/\/)?(?:www\.)?[\w.-]+\.[a-zA-Z]{2,}(?:\/\S*)?$/;
  return regex.test(str);
};

- `^` - начало строки
- `(?:https?:\/\/)?` - необязательная секция, соответствующая протоколу `http://` или `https://`

function setText(message) { 
text = message;
 }
 var text = 'Текст';
 setText('Сообщение');
Опишите, почему получился такой результат.

Ответ: 

Значение переменной text будет равно 'Сообщение' после выполнения кода.

Так как функция setText принимает параметр message и присваивает его значение переменной text,
которая находиться внутри области видимости функции setText и данная функция присваивает переменной text парамент message,
а то что находиться вне области видимости функции она не видит.
А то что переменной text присваивается значение = 'Текст', это происходит в основной части кода
(Глобальной области видимости и никак не влияет на область видимости функции)
